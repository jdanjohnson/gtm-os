# Phase Rules — `design`

Goal: end this phase with a sharp ICP, a falsifiable hypothesis, and a real prospect list
(or at least the recipe to pull one).

## Must-have outputs

- ICP: title + seniority + company shape (B2B) **or** location + industry + business
  shape (local SMB).
- Hypothesis: one sentence, falsifiable, names a concrete metric and target.
- Prospect list (or pullable recipe): size, source, fields.

## Don't transition to `build` until…

- The ICP names buyer authority. ("VP Sales" not "marketer".)
- The hypothesis names a measurable target.
- The list (or recipe) is saved to memory.

## Common mistakes to avoid

- Picking a too-broad ICP ("SaaS companies"). Narrow it.
- Skipping memory search and re-designing what's already been tried.
- Committing to a list size before checking if the tool can return that many ICP-fit rows.
