# Phase Rules — `learn`

Goal: codify what happened so the next experiment is better, not a re-run.

## Must-have outputs

- At least one `memory_save(type="learning", ...)` with confidence reflecting how
  reproducible you believe the result is.
- 1–3 proposed follow-up experiments, each testing a new variable.

## Don't transition to `complete` until…

- At least one learning is saved.
- Follow-ups are listed (or you've concluded there's nothing more to learn here, with a
  reason).

## Memory hygiene

- Confidence ≥ 0.8 with 3+ reinforcements will get promoted to a standing rule
  automatically — write at that confidence only when you're sure.
- Prefer "tactical" learnings over "strategic" ones. "Subject lines that mention a
  specific dollar amount get 22% higher reply rate on at-risk-renewal pitches" is more
  useful than "personalization works".
